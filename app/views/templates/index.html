System Architecture & Pipeline Flow

Deep dive into the KrishiKantho AI multi-agent audio diagnostic infrastructure.

End-to-End Computational Pipeline

1

Audio Ingestion

Accepts farmer's speech inputs via live microphone recording (WebRTC / MediaRecorder API) or pre-recorded audio files.

Tech: PCM WAV mono @ 16kHz
2

STT (Whisper ASR)

Transcribes regional dialects or standard Bangla audio speech inputs to textual Bengali. Incorporates model scale fallbacks.

Fallback: large-v3 → medium → small
3

AI Text Correction

Cleans semantic ASR transcription mistakes, normalizes dialect variances, and produces standard English translation contexts.

Tech: LLM Correction & Repair
4

Weather Context

Fetches local environmental elements (humidity, temp, wind speed) from current coordinates or district indices.

Tech: Open-Meteo Weather APIs
5

BRRI/BARC RAG

Queries vector database matching crop type & symptom tags to extract highly accurate peer-reviewed expert manuals.

Tech: ChromaDB Vector Store
6

AI Advisory & Audit

Processes RAG knowledge sheets alongside weather variables and history context, compiling safe, localized diagnostics.

Tech: Gemini 1.5 / Fallbacks
7

TTS Synthesis

Translates text advice to high-clarity voice output in localized accent tones for low-literacy agricultural users.

Tech: Google TTS / Web Speech API
8

Visual Dashboard

Interactive dashboard reporting diagnostic results, weather stats, historical records, and detailed judge latencies.

Tech: HTML5, CSS Variables, JS
